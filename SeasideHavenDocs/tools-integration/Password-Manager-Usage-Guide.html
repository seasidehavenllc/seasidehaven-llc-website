Password Manager Usage Guide
Purpose

This guide explains how employees and contractors of Seaside Haven LLC must use the approved password manager to securely store, share, and manage credentials. Proper usage ensures protection of sensitive data and compliance with company security standards.

Account Setup

• Install the password manager on approved devices
• Create your account using company email
• Enable multi‑factor authentication (MFA)
• Set a strong master password:
– Minimum 12 characters
– Mix of letters, numbers, symbols
– No personal information
• Store master password securely (never written down)

Storing Credentials

• Store all work‑related passwords in the password manager
• Use folders to organize credentials by division
• Add notes only when necessary (no sensitive data)
• Never store passwords in browsers or unsecured apps
• Update entries when passwords change

Password Requirements

• Minimum length: 12 characters
• Must include uppercase, lowercase, numbers, symbols
• Must be unique for each system
• Must not be reused across platforms
• Must not contain personal information
• Must be changed immediately if compromised

Sharing Credentials

• Use built‑in secure sharing features
• Never send passwords via text or email
• Share only with authorized personnel
• Remove access when users leave the company
• Use “view only” permissions when possible

Security Rules

• Lock the password manager when not in use
• Do not install the password manager on personal devices without approval
• Report suspicious activity immediately
• Do not store personal passwords in company vaults
• Do not bypass MFA or security prompts

Emergency Access

• Admins may grant temporary access for emergencies
• All emergency access must be logged
• Access must be revoked after issue resolution
• Emergency access must follow company security protocols

Audit Requirements

• Password vaults must be reviewed quarterly
• Weak passwords must be updated immediately
• Access permissions must be audited monthly
• All shared credentials must be verified
• Non‑compliant entries must be corrected
