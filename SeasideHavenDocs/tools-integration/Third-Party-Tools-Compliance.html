Third‑Party Tools Compliance
Purpose

This compliance document outlines the rules, requirements, and standards for using third‑party software, platforms, and tools within Seaside Haven LLC. These guidelines ensure security, licensing compliance, and responsible integration across all divisions.

Approved Third‑Party Tools

• Scheduling & Dispatch Tools
• CRM Platforms
• Accounting & Bookkeeping Software
• Payment Processing Systems
• Cloud Storage & Document Management
• Communication & Collaboration Tools
• Security & Encryption Tools
• Division‑Specific Operational Tools

Compliance Requirements

• All tools must be approved by management before use
• Licenses must be valid and up‑to‑date
• Tools must comply with data protection standards
• Access must be restricted to authorized personnel
• Tools must not store sensitive data insecurely
• All integrations must use encrypted communication
• Tools must not violate vendor terms of service
• Usage must align with company security policies

Data Protection Rules

• No customer data may be stored outside approved systems
• Sensitive data must be encrypted at rest and in transit
• Third‑party tools must not share data with external parties
• Tools must support secure authentication (MFA preferred)
• Data retention must follow company guidelines

Access Control

• Access must be granted only as needed
• Accounts must be removed when employees leave
• Shared accounts are prohibited
• Passwords must be stored in the approved password manager
• Admin access must be logged and monitored

Risk Management

• Tools must be reviewed for security risks
• Vulnerabilities must be reported immediately
• Tools with known security issues must be discontinued
• Backup plans must exist for critical tools
• Vendor reliability must be evaluated annually

Vendor Requirements

• Vendor must provide security documentation
• Vendor must support encrypted communication
• Vendor must offer reliable uptime and support
• Vendor must comply with U.S. data protection laws
• Vendor must disclose breach notifications promptly

Audit Requirements

• Tools must be reviewed quarterly
• Licensing must be verified annually
• Access logs must be checked for anomalies
• Compliance issues must be documented
• Non‑compliant tools must be removed immediately
